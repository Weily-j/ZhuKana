## Context

台灣人學日文常卡在五十音，現有解法（背表格、羅馬字輸入法）皆無法有效建立注音與假名的發音連結。本設計以 PWA 形式打造注音九宮格日文鍵盤，利用台灣人已有的注音肌肉記憶作為橋樑，在實際輸入情境中自然學習假名。

**Phase 1 範圍**：純鍵盤工具（ひらがな＋カタカナ輸入），無帳號、無後端、無漢字轉換。
**Phase 2**（本設計不涵蓋）：漢字候補轉換、學習模式、進度追蹤。

## Goals / Non-Goals

**Goals:**
- 台灣使用者能用注音符號直覺輸入平假名與片假名
- 撥動時即時顯示注音↔假名對應，降低學習門檻
- 輸入結果可複製貼至其他 App（旅遊情境）
- PWA 可加至手機主畫面，接近原生 App 體驗
- 介面文案以繁體中文為主，日文術語最小化

**Non-Goals:**
- 系統層級輸入法（PWA 技術限制，無法替換系統鍵盤）
- 漢字轉換（Phase 2）
- 使用者帳號與學習進度（Phase 2）
- 桌面版優化
- 拗音自動組合（きゃ等，使用者手動組合）

## Decisions

### D1：技術選型 — Vanilla JS PWA（非框架）

**決定**：使用 HTML/CSS/Vanilla JS，不引入 React/Vue 等框架。
**理由**：
- 鍵盤核心邏輯（觸控事件、組字緩衝）不需要元件化框架
- 減少 bundle size，提升手機載入速度
- PWA manifest + Service Worker 即可達成離線使用
**替代方案**：Vue 3（熟悉度高但 bundle 較大），React（過度工程化）

### D2：撥動方向 — 沿用日文標準フリック

**決定**：點按＝あ段，上↑＝い段，右→＝う段，下↓＝え段，左←＝お段。
**理由**：
- 使用者學會後可無縫銜接原生日文鍵盤
- 減少重新學習成本
**替代方案**：依注音母音順序重排（ㄚㄛㄜㄝ），但會造成與日文標準不一致

### D3：例外字處理 — 撥動時橘色高亮（Progressive Disclosure）

**決定**：鍵盤靜態時例外行按鍵顯示橘色邊框，撥動展開時例外字格子橘色高亮並顯示正確注音組合。
**理由**：
- 保持鍵盤介面整潔，不在靜態狀態顯示過多資訊
- 在使用者最需要的時機（選字瞬間）提供正確發音提示
**替代方案**：A. 角落小字徽章（資訊不夠明確）；B. 按鍵上直接顯示雙層符號（視覺過擁擠）

### D4：濁音輸入 — 修飾鍵循環（沿用日文標準）

**決定**：輸入清音後按 `變音` 鍵循環：清音 → 濁音(゛) → 半濁音(゜) → 小字 → 清音。
**理由**：與日文原生鍵盤行為一致，使用者只需學一次
**替代方案**：獨立頁面放置濁音行（增加操作步驟，學習曲線更高）

### D5：組字機制 — 前端假名緩衝區

**決定**：Phase 1 使用前端純 JS 實作假名組字緩衝區，不接後端 IME。
**理由**：
- Phase 1 只需假名層（無需漢字轉換），純前端即可實作
- 零後端依賴，離線可用
- Phase 2 再接 Kuromoji.js 或後端 IME 擴充漢字功能
**替代方案**：直接接 Google IME API（增加網路依賴，Phase 1 不必要）

### D6：候補欄 — 靜態詞庫（Phase 1）

**決定**：Phase 1 候補欄使用預建靜態假名詞庫（常用詞 JSON），前端比對。
**理由**：旅遊情境常用詞有限，靜態詞庫足夠覆蓋核心需求，無需後端
**替代方案**：空候補欄（使用者體驗差）；動態後端 API（Phase 1 過度複雜）

## Risks / Trade-offs

| 風險 | 影響 | 緩解策略 |
|------|------|----------|
| PWA 無法作為系統鍵盤 | 使用者需切換 App 才能複製貼上，增加摩擦 | 突出顯示複製按鈕，教學引導說明使用流程 |
| 撥動手勢與捲頁手勢衝突 | 使用者想撥動選字，卻觸發頁面捲動 | 鍵盤區域使用 `touch-action: none`，`preventDefault` 攔截 touch 事件 |
| iOS Safari PWA 限制 | Vibration API 在 iOS 不支援 | 降級處理，無震動但不影響功能 |
| 注音↔假名對應學習曲線 | 初次使用者不理解注音符號與假名的對應 | Onboarding 說明頁（可跳過），撥動時注音提示 |
| 例外字發音記憶負擔 | し/ち/つ/ふ 等例外字與注音不一致，容易混淆 | 橘色高亮＋正確注音組合在選字時即時顯示 |

## Migration Plan

本次為全新 PWA，無現有系統需遷移。

部署方式：靜態檔案部署（Vercel / GitHub Pages / Netlify），無後端伺服器。
離線支援：Service Worker 快取所有靜態資源，首次載入後可完全離線使用。
回滾策略：版本控制，前端靜態部署可即時回滾至前一版本。

## Open Questions

1. **拗音輸入流程**：きゃ 等拗音目前需要使用者手動操作（き → 變音 → ゃ），是否需要在 Phase 1 提供快捷方式？
2. **長音符 ー 的按鍵位置**：目前規劃放在 `。、?!` 鍵的撥動方向之一，需確認最自然的撥動方向。
3. **候補詞庫大小**：靜態詞庫包含多少詞較合適？建議以旅遊常用詞為核心（500–1000 詞）。
